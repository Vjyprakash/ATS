🗄️ Connect a Database
Right now, all data lives in your browser's localStorage — it's private and fast, but it's only on this one device and can be cleared. To share data across your team and persist everything reliably, connect a real database using one of these options:
🔥
Firebase / Firestore
Free tier
Google's real-time NoSQL database. Easy to set up, works directly from the browser — no backend needed. Perfect for small teams. Recommended for most users.
firebase.google.com →
⚡
Supabase (PostgreSQL)
Free tier
Open-source Firebase alternative backed by Postgres. Has a REST API and JS client — works from the browser. Great for structured data and SQL queries.
supabase.com →
🍃
MongoDB Atlas
Free tier
Cloud MongoDB with a Data API that accepts HTTPS calls. JSON-friendly — each candidate is already a document. Needs a small server-side proxy for security.
mongodb.com/atlas →
📊
Airtable
Paid plans
Spreadsheet-style database with a REST API. No SQL needed — great if your team prefers a visual grid view alongside this app. Has a free tier for small bases.
airtable.com →
🖥️
Self-hosted (MySQL / PostgreSQL)
Self-hosted
Run your own database on a VPS or server. Requires a small backend API (Node.js / Python / PHP) to bridge this app to the DB. Most secure — your data never leaves your infrastructure.
💡 How to integrate: Use the Export CSV feature to migrate your existing candidates to any database. Then add a few lines of JavaScript to replace the save() and load() functions in this app to call your chosen database API instead of localStorage. Ask Claude for help wiring it up!
⚡ How to Connect Supabase
Step 1 — Create a free project
Go to supabase.com → New Project. Note your Project URL and anon public key from Project Settings → API.

Step 2 — Create the candidates table
In Supabase → SQL Editor, run:
CREATE TABLE candidates (
  id TEXT PRIMARY KEY,
  data JSONB NOT NULL,
  updated_at TIMESTAMPTZ DEFAULT now()
);
Enable Row Level Security → add a policy: true for anon read/write (or authenticated-only for production).

Step 3 — Replace save() / load() in this file
Open TalentFlow_ATS.html in a text editor. Find the save() and load() functions and replace with:

⏰ Daily Auto-Scan (9:30 PM)
Browsers cannot read folders automatically for security reasons — but you can run a tiny local Python script every evening that watches a folder and uploads new CVs via this app. Here's the full setup:
Option A — Python watcher script (Windows/Mac/Linux)
Save this as scan_cvs.py in any folder: 
import os, glob, datetime

# ← Change this to your daily CV drop folder
CV_FOLDER = r"C:\Users\YourName\Downloads\CVs"

def get_todays_files():
    today = datetime.date.today().strftime("%Y-%m-%d")
    all_files = glob.glob(os.path.join(CV_FOLDER, "*.pdf")) + \
                glob.glob(os.path.join(CV_FOLDER, "*.docx"))
    # Filter files modified today
    return [f for f in all_files
            if datetime.date.fromtimestamp(os.path.getmtime(f))
               .strftime("%Y-%m-%d") == today]

files = get_todays_files()
print(f"Found {len(files)} CVs for today:")
for f in files: print(" -", os.path.basename(f))
# → Then call your Supabase API or open the browser to upload
Schedule it with Windows Task Scheduler (search "Task Scheduler" → Create Task → Trigger: Daily 21:30) or cron on Mac/Linux:
30 21 * * * python3 /path/to/scan_cvs.py
Option B — Use this app's Upload page
Save your daily CVs to a fixed folder (e.g. C:\CVs\2025-01-15\). Each evening at 9:30 PM, open TalentFlow → Upload CVs → drag that folder in. The app processes all files and adds new candidates automatically.
💡 Pro tip: Name your download folders by date — e.g. CVs_2025-01-15 — so you can re-upload any past day's batch easily.
